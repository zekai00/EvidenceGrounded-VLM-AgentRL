## 项目拆分

建议把 `/root/Workspace/VLM` 下的两个方向拆成两个独立 GitHub 仓库：

| 本地目录 | GitHub 仓库建议 | 主线 |
|---|---|---|
| `/root/Workspace/VLM/EviTool-VL` | `EviTool-VL` | Browser / GUI tool-call RL，Playwright 环境，BrowserRL SFT/RL |
| `/root/Workspace/VLM/EvidenceGrounded-VLM-AgentRL` | `EvidenceGrounded-VLM-AgentRL` | 多模态文档主动取证，EvidenceGrounded trajectory SFT / verifier-guided RL |

不要把两个目录放进同一个 Git 仓库。原因：

- 任务定义不同：一个是 GUI/browser action agent，另一个是 evidence-seeking document agent。
- 数据和实验产物不同：BrowserRL 的 task suite / rollout，与 EvidenceGrounded 的 PDF page/evidence index 不应混在一起。
- 简历和面试叙事不同：两个项目分别能讲清楚，不会互相稀释。

## 创建远程仓库

如果用 GitHub 网页：

1. 在 GitHub 新建仓库：`EvidenceGrounded-VLM-AgentRL`。
2. 建议先设为 private，整理好 README 和报告后再决定是否 public。
3. 不要勾选自动生成 README / .gitignore / license，因为本地已经有。
4. 创建后在本地执行：

```bash
cd /root/Workspace/VLM/EvidenceGrounded-VLM-AgentRL
git remote add origin git@github.com:zekai00/EvidenceGrounded-VLM-AgentRL.git
git push -u origin main
```

如果之后安装并登录 `gh`：

```bash
cd /root/Workspace/VLM/EvidenceGrounded-VLM-AgentRL
gh repo create zekai00/EvidenceGrounded-VLM-AgentRL --private --source=. --remote=origin --push
```
